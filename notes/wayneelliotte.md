# 2026-05-28

传统隐私观念 (防守型) ──> 保护隐私就是隐藏信息、防止老大哥监视 新密码朋克观念 (进攻型) ──> 隐私是拥有数据的自主权，是构建去中心化社会的信任根基

ZKP 是 Zero-Knowledge Proofs（零知识证明） 的缩写。证明者（Prover）能够在不向验证者（Verifier）透露任何有用信息（隐私）的前提下，使验证者相信某个断言是正确的。

一个合法的零知识证明系统必须满足以下三个条件：

完备性（Completeness）： 如果断言是真的，只要证明者诚实，她就一定能通过验证者的测试（真金不怕火炼）。

可靠性（Soundness）： 如果断言是假的，那么任何作弊的证明者，在概率上都几乎不可能欺骗成功（骗子终究会露馅）。

零知识性（Zero-Knowledge）： 证明过程结束后，验证者除了知道“这个断言是真的”这一事实外，无法获得任何其他额外的信息（隐私保密）。

全同迹加密 (FHE)

让加密数据在无需解密的情况下直接进行计算，彻底解放云端隐私。

传统的银行账户和信用卡高度依赖人类的生物信息（如刷脸、短信验证码 2FA）。如果一个 Agent 自动帮你在网上订机票，它卡在输入短信验证码这一步，自动化流程就会中断。Agent 支付产品通过编程接口（API）为 AI 创建了机器数字身份。它利用加密货币钱包（Web3 基础设施）或新型的数字化银行账户（如虚拟信用卡/VCC），让 Agent 可以通过代码签名、公私钥校验等机器可读的方式，合规、自主地完成身份验证并进行支付。

Agent 之间的交互往往是高频、微额的。如果走传统的 Visa/Mastercard 渠道，每笔交易的固定手续费（如 $0.3）远超交易本身的价值。大量采用区块链 Layer 2（如 Base, Arbitrum）、闪电网络（Lightning Network）或状态通道（State Channels）。将单笔交易手续费降低到分币级（$0.001 甚至更低）。支持每秒数万次的并发微支付，满足 AI 毫秒级调用 API 时的实时结算需求。

agent 支付最核心的风险控制（Risk Control）模块。产品提供了极其精细化的策略引擎（Policy Engine）：

-   **单次/每日限额**：限制 Agent 每天最多只能花 $50。
    
-   **白名单商户**：限定该钱包只能向特定的 API 供应商（如 OpenAI, Serper）付款，禁止向其他未知地址转账。
    
-   **按条件触发（Conditional Auth）**：小额（如 $5 以内）允许 Agent 自动决策，大额交易（如 $100 以上）自动挂起并向人类发送 Telegram/邮件通知，等待人类“一键确认”

LLM的机制、训练和架构

基本机制是数学算法预测一段文本后面出现的词。给可能的词分配概率。可能会选择概率没那么高的词，所以回答会不一样。

预训练是阅读海量文本。所以gemini和gork其实很有优势。模型行为由成千上万的参数权重决定，一般也不会去认为调整。而是通过反向传播Backpropagation的方式使预测值靠近真实值。训练非常消耗算力

架构，google发明了transformer，把一段文本的词转换成数学向量，可以并行运算适配GPU，让文本不用一字一字的读取。

attension机制是transformer的灵魂，让词语之间产生联系，根据语境调整词义。Emergent Phenomenon涌现现象，这套机制作出精准的预测
