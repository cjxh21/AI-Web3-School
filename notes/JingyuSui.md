\### 🗓️ 每日详细指南

\#### **🔵 周二：启动日 - 评估基础与搭建环境**

\* **判断基础**：花15分钟诚实回答：提到“API”、“私钥”、“合约”，哪个让你感觉更陌生？这决定了你本周的\*\*主要精力\*\*是倾斜给模块A（AI）还是模块B（Web3）。

\* **主线任务**：

1\. **搭建 Learning Agent**：选择一个工具开始。如果你已有 **ChatGPT/Claude 会员**，直接使用它们；否则，根据课程指引去 **Z.ai/GLM** 平台申请 API Key 并接入 **Codex/Claude Code**。

2\. **下达第一个指令**：把这份学习计划发给你的 Agent，对它说：“请根据这份计划，帮我生成周三学习 LLM 的详细大纲，并解释 Prompt, Workflow, Agent 三者的核心区别。”

3\. **创建 GitHub 仓库**：命名为 `AI-Web3-Learning-Journey`，创建好 `notes`, `demos`, `prompts` 等文件夹，并把仓库链接和 Agent 配置截图保存好。

\#### **🟢 周三：AI 基础 - 理解模型与“智能体”**

\* **学习重点**：LLM 的“概率生成”本质，以及 **Prompt (单次问答) -> Workflow (固定流程) -> Agent (自主决策)** 这条复杂度光谱。

\* **实践任务**：

1\. 观看推荐材料中的视频 `What is a Large Language Model?`。

2\. 使用你的 Agent，让它帮你生成一份 **“Prompt vs Workflow vs Agent 对比表”**，并附上典型应用场景。

3\. **关键思考**：在你的笔记中回答——“为什么高合规场景下，不建议过度 Agent 化？”

\#### **🟠 周四：Web3 基础 - 拥有你的第一个钱包**

\* **学习重点**：钱包、账户、私钥、助记词的本质区别。牢记\*\*安全铁律：助记词和私钥绝不能泄露、截图或上传到网络\*\*。

\* **实践任务**：

1\. 安装 **MetaMask** 钱包，安全备份助记词（推荐纸笔记录）。

2\. 切换到 **Sepolia 测试网**，并搜索 “Sepolia Faucet” 领取测试币。

3\. 在你的笔记中，用比喻解释：“地址是你的用户名，私钥是你的密码，助记词是恢复密码的密钥，签名是用你的密码授权一个操作。”

\#### **🟣 周五：深入实践 - API 调用与链上交互**

\* **AI 实践**：参考 [**Z.ai**](http://Z.ai) **API 文档** 或 **Anthropic API 文档**，用 **Postman** 或直接在 **Colab** 里写一小段代码，成功调用一次 API，感受 `temperature`, `max_tokens` 等参数。

\* **Web3 实践**：

1\. 用 MetaMask 给你自己创建的另一个钱包地址发送 **0.001 SepoliaETH**。

2\. 去 **Etherscan (Sepolia 测试网版本)**，搜索这笔交易，查看它的状态、Gas 费、区块高度。

3\. 保存好 **交易哈希 (Transaction Hash)**，这是你链上活动的凭证。

\#### **🚀 周末：交叉实验 - 连接 AI 与 Web3 的第一次握手**

这是本周最有价值的环节，把两个领域串联起来。

\* **实验任务**：让 AI 指导你部署一个智能合约，但\*\*你作为人类，负责复核和安全确认\*\*。

\* **具体步骤**：

1\. 对你的 Agent 说：“请提供一个最简单的 Solidity 智能合约（比如一个存数字的合约），并给出通过 Remix IDE 部署到 Sepolia 测试网的步骤。”

2\. 打开 **Remix IDE**，粘贴代码。\*\*仔细检查\*\*，思考合约的用途。

3\. 在 MetaMask 中确认部署交易。\*\*这时，你是最终的决策者。\*\*

4\. 部署成功后，调用合约的读写函数。画出流程图：“\*\*AI 生成代码 -> 你复核 -> 钱包确认 -> 链上执行 -> 区块浏览器验证\*\*”。

5\. **关键记录**：记下合约地址、部署交易的哈希，并在笔记中写下你对“AI 生成代码安全性”的思考。

\#### **🔴 周一：复盘与沉淀 - 从实践到认知**

\* **交付物整理**：将所有截图、笔记、代码、交易哈希、流程图整合到你的 **GitHub 仓库**，并编写清晰的 `README.md`。

\* **概念说明**：写一段300字以内的文字，把 **LLM, Agent, 钱包, 签名, 交易, 合约** 串在一个“AI 代理帮你管理链上身份”的假想任务里解释。

\* **信息源建立**：在 Twitter (X) 或 Farcaster 上，关注 3-5 个提及 **AI × Web3 / Crypto AI** 的开发者或项目，为下周的“行业观察”任务做准备。

\### 💡 几个关键提醒

\* **安全永远是第一位**：在整个学习过程中，反复锤炼安全意识。记住，\*\*AI Agent 不应接触你的私钥\*\*，涉及签名、转账的操作，保留人工做最终确认的环节。

\* **拥抱不确定性**：本周的任务可能不会一帆风顺，测试网拥堵、代码报错都是常事。将解决问题的过程记录下来，这本身就是宝贵的学习材料。

\* **善用 Agent**：你是 Agent 的指挥者，让它帮你查资料、解释报错、生成笔记、起草代码，而不是代替你思考。把省下的时间，用在理解核心概念和做决策上。

一周后，你不仅会收获一堆概念和一串链上记录，更会建立一套“AI辅助 + 人工确认 + 链上验证”的思维框架。祝你第一周学习顺利！如果有具体问题，可以随时再来交流。
