## Packed Authority

是一個建立在錢包之上的基礎執行層授權協議，專門用來限制和規範 AI Agent 的鏈上操作邊界，明確告訴 Agent **「能做什麼、不能做什麼，以及在什麼條件下必須停止操作。**  

一個完整的 Pack 通常包含以下四個核心要素：

1.  **Intent (意圖)：** 規定 Agent 被授權完成的具體目標，例如「當以太坊低於 2000 美元時買入」。
    
2.  **Execution Plan (執行計畫)：** 將意圖轉化為具體的行動細節，明確指出 Agent 需調用哪個合約、操作的代幣數量及交易對，形成一份透明且可審計的計畫。
    
3.  **Policy (風控約束)：** 這是 Pack 最核心的部分，涵蓋總預算限制、審批白名單、合約限制，甚至可精確到 ABI 參數級別的約束。每次執行時，策略引擎都會強制校驗以確保不越界。
    
4.  **Completion Condition (完成條件)：** 設定授權的失效條件（例如達到 1000 美元的消費上限或特定的時間節點），條件觸發後會自動撤銷權限並攔截後續交易
    

## Recipe（技能層

-   **Uniswap V3：** 系統內建了 Uniswap V3 的標準執行規範與路由 (Router) 合約地址。這能確保 Agent 在進行代幣兌換 (Swap) 時，嚴格遵循正確的查詢與兌換步驟（例如固定的 step 1, 2, 3, 4），防止模型產生幻覺而去調用錯誤或被竄改的合約路徑。
    
-   **PY Market：** 支援此類市場的標準交互規範，使 Agent 具備在該特定平台上操作的能力。
    
-   **Hyperliquid：** 支援該去中心化交易所的相關合約，確保 Agent 能在可控邊界內執行複雜的訂單與交易場景。
    

**這些預設合約在 Recipe 中的具體作用：** Recipe 會將執行這些主流 DeFi 任務所需的**正確合約地址、ABI 參數以及安全風控需要參考的條件，全部打包封裝在一個檔案中**。當用戶的 Agent 需要在這些平台上進行交易時，會直接調用對應的 Recipe 來學習應如何高效且準確地發起合約調用，而不是依賴大語言模型自身的知識庫臨場發揮

思考錢包和ai交易的交叉

Human-in-the-loop。  
AI 可以準備交易，但最後由人確認。

常見場景：

-   第一次付款給新地址
    
-   超過限額
    
-   跨鏈轉帳
    
-   授權智能合約
    
-   涉及高價資產
    
-   AI 判斷信心不足
    

**子錢包機制**  
不要讓 AI 使用主錢包，而是給它一個獨立子錢包。

好處：

-   只放少量資金
    
-   降低主資產風險
    
-   容易停用
    
-   方便追蹤 AI 的支出
    
-   不同 AI Agent 可使用不同錢包

# 今天將Hemers安裝到外接硬盤上

首先在外接硬盤上裝上WSL

然後配置環境

```
wsl -d Hermes-Ubuntu-22.04
```

安裝並驗證

```
hermes --version
```

並安裝好Node v22.22.3、npm 10.9.8、ripgrep、ffmpeg，把 hermes 固定到 /usr/local/bin/hermes。我也關掉了這個 WSL 的 Windows PATH 繼承，避免 Windows 路徑裡的括號干擾 Linux 指令。  
  
下一步配置API Key  

```
wsl -d Hermes-Ubuntu-22.04
```

然後在WSL裡  

```
hermes setup
```

最後啟動

```
hermes
```
