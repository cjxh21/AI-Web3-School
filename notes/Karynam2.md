# 5月25日  
《Agent Memory》

🧠 一句话总结

智能体记忆不只是“记住你说了啥”，而是让智能体在不同的时间、任务、会话中，能记得“之前干到哪了、你想干啥、该怎么做”，从而继续干活，而不是每次都从头来。

1️⃣ 记忆是“跨时间的意图和状态管理”

用户一开始说的往往不是完整任务，只是个方向。

智能体通过干活（问问题、做决定、用工具）慢慢形成“状态”。

记忆的作用就是：把这种状态带到下一次对话或任务中，让工作能继续。

大白话：你说“帮我写个报告”，它得记得你上次说“要PPT风格”，而不是每次都要你重新说。

2️⃣ 产品角度看记忆：别让我重新说一遍

用户已经说了目标、做了决定，下回应该接着来，而不是重头问。

好的记忆让体验像“继续”，而不是“重启”。

例子：ChatGPT 记得你喜欢的语气、你的名字、你常用的格式。

3️⃣ 工程角度看记忆：要能写、能查、能改

记忆不是“存一堆聊天记录”，而是要：

写：谁写的？用户？智能体？文件？

存：存成啥样？向量？结构化？

查：当前任务需要哪些记忆？

改：旧信息错了怎么办？

大白话：不是所有话都值得记，记了也不一定每次都翻出来，错了还得能改。

4️⃣ 两种典型产品的记忆方式

产品 记什么 为什么

ChatGPT 用户偏好、语气、个人信息 让对话更顺、少重复

Claude Code 项目命令、代码规范、之前失败的经验 让编程任务能接着干

大白话：聊天记“你是谁”，写代码记“项目怎么搞”。

5️⃣ 记忆的两种维度

时间轴：短期（当前会话） vs 长期（跨会话）

内容轴：

经历型（之前发生过啥）

事实型（用户偏好、项目规则）

流程型（怎么做某件事）

大白话：有些事记一阵子，有些事记一辈子，有些是“怎么做事”的流程。

6️⃣ 记忆的生命周期

不是所有历史都值得记

不是所有记下的都值得翻出来

旧记忆要能更新或删除

大白话：记忆不是垃圾桶，是工具箱。

7️⃣ 记忆在技术栈中的位置

提示工程：这轮怎么问

上下文工程：这轮能看到哪些记忆

控制工程：工具、循环、日志、策略怎么跑

大白话：记忆不是独立存在的，它影响“看到啥”“怎么做”。

8️⃣ 行业方向：记忆正在成为基础设施

产品：mem0（通用记忆层）

平台：Letta（有状态的智能体）

框架：LangMem、Zep、EverOS（记忆操作系统）

大白话：大家都在做“让智能体记住事”的基础设施。

✅ 最终结论

产品层：记忆让体验顺、少重复、能接着干

工程层：记忆需要能写、能查、能改、能忘

战略层：记忆不是功能，是智能体的基础设施
