# 學習筆記

### Handbook《大語言模型（LLM）》學習筆記

**1\. 核心認知（第一性原理）**

-   **LLM 是機率模型，而非真相源**：模型輸出的是機率上合理的內容，而不是天然可信的事實。
    
-   **定位為推理層**：模型可用於理解、歸納、生成和規劃，但不能單獨承擔事實來源與權限判斷。關鍵事實必須來自資料庫、API 等外部可信系統。
    
-   **暴露不確定性**：當資訊不足或過期時，系統設計應讓其顯式降級，而不是讓模型編造順滑的答案。
    

**2\. 五大關鍵底層概念**

-   **Token（詞元）**：模型處理文本的基本單位。影響上下文容量與成本，需注意程式碼、JSON 和表格通常比普通文字消耗更多 Token。
    
-   **Embedding（嵌入）**：將內容映射為向量以衡量「語義接近度」，常用於檢索增強生成（RAG）。但語義相近不代表結論正確，不能替代來源校驗。
    
-   **Transformer 架構**：依靠注意力機制（Attention）在上下文中尋找模式，賦予模型強大的組合能力，但**沒有給予它事實的最終裁決權**。
    
-   **Hallucination（幻覺）**：模型會生成看似合理卻不真實的內容（如編造 API 或假資料）。防範幻覺不能只靠優化提示詞，必須依賴外部規則檢查與人工確認。
    
-   **Multimodal（多模態）**：使模型能讀懂圖表、截圖等真實工作介面，但關鍵判斷仍需回歸結構化的數據與可信來源。
    

**3\. AI x Web3 的系統實踐原則**

-   LLM 位於系統的「理解和生成層」，需與資料層、編排層、執行層和安全層配合。
    
-   **核心守則**：LLM 越靠近執行層，就越需要將其自然語言輸出轉化為「可驗證的物件」（如 schema、參數、日誌），以確保系統運作的安全與穩定。
