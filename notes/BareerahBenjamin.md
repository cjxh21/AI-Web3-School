注意力机制按简单到复杂分为：

简化的自注意力：一种简化的自注意力技术，用于引入更广泛的概念

自注意力：带有可训练权重的自注意力机制，这是大语言模型所使用机制的基础

因果注意力：大语言模型使用的一种自注意力机制，该机制允许模型只关注序列中先前和当前的输入，从而保持文本生成的时间顺序。在自注意力的基础上增加了额外掩码，使得大语言模型可以一次生成一个单词。

多头注意力：自注意力和因果注意力的一种扩展，该机制使模型能够同时关注来自不同表示子空间的信息，它将注意力机制划分成多个头，从而使模型能够并行捕获输入数据的各种特征。

编码器-解码器 RNN 的主要限制：在解码阶段，RNN 无法直接访问编码器中的早期隐藏状态。因此，它只能依赖当前的隐藏状态，这个状态包含了所有相关信息。这可能导致上下文丢失，特别是在复杂句子中，依赖关系可能跨越较长的距离。于是开发 Bahdanau 注意力机制（使得解码器在每个解码步骤中可以选择性地访问输入序列的不同部分）。

在计算序列表示时，自注意力机制允许输入序列中的每个位置关注同一序列中的所有位置。自注意力机制是基于 Transformer 架构的当代大语言模型（如 GPT 系列）的关键组成部分。

在自注意力机制中，“自”指的是该机制通过关联单个输入序列中的不同位置来计算注意力权重的能力。

在自注意力机制中，我们的目标是为输入序列中的每个元素 x 计算上下文向量 z 。上下文向量（context vector）可以被理解为一种包含了序列中所有元素信息的嵌入向量。它们的目的是通过结合序列中其他所有元素的信息，为输入序列（如一个句子）中的每个元素创建丰富表示。

实现简单自注意力机制的步骤：

第一步：计算中间值 w，即所谓注意力分数。通过计算查询词元 x 与其他所有输入词元的点积来确定这些分数。

第二步：对先前计算的每个注意力分数进行归一化处理，其主要目的是获得总和为 1 的注意力权重。这种归一化是一个惯例，有助于解释结果，并能维持大语言模型的训练稳定性。

在实际应用中，使用 softmax 函数进行归一化更为常见，这种方法更好地处理了极值，并在训练期间提供了更有利的梯度特性。另外，softmax 函数可以保证注意力权重总是正值，这使得输出可以被解释为概率或相对重要性，其中权重越高表示重要程度越高。

第三步：通过将嵌入的输入词元 x 与相应的注意力权重相乘，再将得到的向量求和来计算上下文向量 z 。因 此，上下文向量 z 是所有输入向量的加权总和，通过将每个输入向量与其对应的注意力权重相乘而获得。

带可训练权重的自注意力机制：在原始 Transformer 架构、GPT 模型和大多数其他流行的大语言模型中使用的自注意机制，同时它也被称为缩放点积注意力。最显著的区别是这里引入了在模型训练期间更新的权重矩阵。可训练的权重矩阵用于查询向量、键向量和值向量:

缩放点积注意力的原理：对嵌入维度进行归一化是为了避免梯度过小，从而提升训练性能。例如，在类 GPT 大语 言模型中，嵌入维度通常大于 1000，这可能导致点积非常大，从而在反向传播时由于 softmax 函数的作用导致梯度非常小。当点积增大时，softmax 函数会表现得更像阶跃函数，导致梯度接近零。这些小梯度可能会显著减慢学习速度或使训练停滞。因此，通过嵌入维度的平方根进行缩放解释了为什么这种自注意力机制也被称为缩放点积注意力机制。

因果机制：调整注意力机制，防止模型访问序列中未来的信息，每个词的预测只能依赖之前出现的词。

多头机制：涉及将注意力机制分成多个“头”。每个头会学习数据的不同特征，使模型能够在不同的位置同时关注来自不同表示子空间的信息。

因果注意力：也称为掩码注意力，是一种特殊的自注意力形式。它限制模型在处理任何给定词元时，只能基于序列中的先前和当前输入来计算注意力分数，而标准的自注意力机制可以一次性访问整个输入序列。

在因果注意力中，获得掩码后的注意力权重矩阵的一种方法是对注意力分数应用 softmax 函数，将对角线以上的元素清零，并对所得矩阵进行归一化。

在因果注意力中，获得掩码后的注意力权重矩阵的一种更有效的方法是在应用 softmax 函数之前将注意力分数用负无穷大值进行掩码：(softmax 函数会将其输入转换为一个概率分布。当输入中出现负无穷大值（–∞）时，softmax 函数会将这些值视为零概率。（从数学角度来看，这是因为 e–∞ 无限接近于 0。）)

dropout 掩码：dropout 是深度学习中的一种技术，通过在训练过程中随机忽略一些隐藏层单元来有效地“丢弃”它们。这种方法有助于减少模型对特定隐藏层单元的依赖，从而避免过拟合。需要强调的是，dropout 仅在训练期间使用，训练结束后会被取消。两个特定时间点使用：一是计算注意力权重之后，二是将这些权重应用于值向量之后。

多头注意力：

① 从因果注意力扩展到多头注意力。首先，将直观地通过堆叠多个 CausalAttention 模块来构建多头注意力模块。然后，将用一种更复杂但计算上更高效的方式来实现这个多头注意力模块。

多头注意力的主要思想是多次（并行）运行注意力机制，每次使用学到的不同的线性投影——这些投影是通过将输入数据（比如注意力机制中的查询向量、键向量和值向量）乘以权重矩阵得到的。在代码中，可以通过实现一个简单的 MultiHeadAttentionWrapper 类来达到这一目标，MultiHeadAttentionWrapper 类堆叠了多个之前实现的 CausalAttention 模块实例。

② 权重划分实现多头注意力，在 MultiHeadAttention 中，计算完注意力权重和上下文向量后，将所有头的上下文向量转置为(b, num\_tokens, num\_heads, head\_dim)的形状。这些向量接着会被重塑（展平）为(b, num\_tokens, d\_out)的形状，从而有效地整合所有头的输出。此外，我们在 MultiHeadAttention 中添加了一个输出投影层（self.out\_proj），这是在合并多个头之后的步骤，而 CausalAttention 类中并不存在这个层。

词嵌入（Text Embedding）

① 嵌入：数据 → 向量，本质是将离散对象映射到连续向量空间中的点，目的是将非数值数据转换为神经网络可以处理的格式。视频、音频和文本格式需要不同的嵌入模型。

句子嵌入或段落嵌入在检索增强生成（retrieval-augmented generation）领域非常流行。通过将生成（如生成文本）与检索（如搜索外部知识库）相结合，检索增强生成能够在生成过程中引入与上下文相关的外部信息。

② word2vec：预训练模型，词嵌入早期最流行的方法。核心思想是出现在相似上下文中的词往往具有相似的含义；实现了根据目标词预测上下文，或根据上下文预测目标词，从而生成词嵌入。

③ 词嵌入的维度越高越能捕捉到更细微的关系，但是计算效率更高。而高维度（>3）嵌入也难以进行可视化。

④ LLM 模型通常会自行生成嵌入，与使用 word2vec 相比，自行生成嵌入层可以针对特定任务和数据进行优化。

移除空白字符可以减轻内存和计算的负担。然而，如果训练的模型需要对文本的精确结构保持敏感，那么保留空白字符就很重要

② 分词器一般包括接收文本样本并将其分词为单独词元后利用词汇表将词元转换为词元 ID 的 encode 方法和接收一组词元 ID 并将其转换回文本词元后将其连接为自然语言文本的 decode 方法。

引入特殊上下文词元

① 为了处理词汇表之外出现的单词，会引入特殊上下文词元(可能包括用于标识未知词汇和文档边界的词元)来增强模型对上下文和其他相关信息的理解：

<|unk|>：表示未出现在训练数据中，未被包含在现有词汇表的新词和未知词

<|endoftext|> ：分隔两个不相关的文本来源

在不同的大语言模型中，研究人员可能会考虑引入如下这些特殊词元：

\- \[BOS\]（序列开始）：标记文本的起点。

\- \[EOS\]（序列结束）：位于文本的末尾，类似<|endoftext|>，特别适用于连接多个不相关的文本。

\- \[PAD\]（填充）：当使用批次大小（batch size）大于 1 的批量数据训练大语言模型时，数据中的文本长度可能不同。为了使所有文本具有相同的长度，较短的文本会通过添加\[PAD\]词元进行扩展或“填充”，以匹配批量数据中的最长文本的长度。

GPT 模型仅使用<|endoftext|>词元来简化其处理流程，其作用与\[EOS\]词元作用相似。此外，<|endoftext|>也被用于文本的填充。后续在批量输入上训练时会采用掩码技术，即具体使用哪种词元来填充实际上不重要。

GPT 模型的分词器也不使用<|unk|>词元来处理超出词汇表范围的单词，而是使用 BPE 分词器将单词拆解为子词单元。

BPE 分词器

用于训练如 GPT-2、GPT-3 和 ChatGPT 的原始模型。可以利用 Python 的 tiktoken 库实现 BPE 算法。

原理：将不在预定义词汇表中的单词分解为更小的子词单元甚至单个字符，从而能够处理词汇表之外的单词：

利用滑动窗口数据采样 使用滑动窗口生成用于训练模型的输入-目标对（数据加载器），步骤如下：给定一个文本样本，我们从中提取子样本，作为输入块提供给大语言模型。在训练过程中，模型的任务是预测输入块之后的下一个词，我们会屏蔽目标词之后的所有单词。必须说明的是，在大语言模型处理文本之前，文本会经过分词处理，但为了更清晰地说明，这里省略了分词步骤。

高效数据加载器实现需要用到 PyTorch：

这个数据加载器会遍历输入数据集，并将输入和目标以 PyTorch 张量的形式返回，这些 PyTorch 张量可以被视为多维数组。我们需要返回两个张量，张量 x 中，其中每行代表一个输入上下文；第二个张量 y 包含相应的预测目标（下一个词），它们是通过将输入移动一个位置创建的。实际操作中会直接操作 token ID 而非 token。

小的 batch\_size 便于演示，也会减少训练过程中的内存占用，但是会导致在模型更新时产生更多噪声(noise)。

max\_length 与 stride 的关系：如果步幅 stride 设置为 1，那么在创建下一个批次时，输入窗口向前移动一个位置。如果步幅 stride 与输入窗口大小max\_length 相等，则可以避免批次之间的重叠。

batch\_size=8, max\_length=4, stride=4：选择将步幅增加到 4 来充分利用数据集（不会跳过任何一个单词），同时避免不同批次之间的数据重叠，因为过多的重叠可能会增加模型过拟合的风险。

创建词元嵌入

① 最后一步，即 token ID → embedding vector。在初始阶段需要用随机值初始化嵌入权重。

② 类 GPT LLM 使用反向传播算法训练深度神经网络，需要连续向量表示或嵌入。

③ 嵌入层的权重矩阵由小的随机值构成。作为模型优化工作的一部分，这些值将在大语言模型训练过程中被优化。嵌入层实质上执行的是一种查找操作，它根据词元 ID 从嵌入层的权重矩阵中检索出相应的行。

本质上可以将嵌入层方法视为一种更有效的实现独热编码的方法。它先进行独热编码，然后在全连接层中进行矩阵乘法，

编码单词位置信息

嵌入层的工作机制是将相同的词元 ID 映射为相同的语义向量。但由于这种映射是位置无关的，且自注意力机制也无法感知顺序，因此模型必须引入额外的位置编码并将其与词元向量相加，才能让模型分辨出词元在序列中的具体位置。

有两种位置信息嵌入策略：

绝对位置嵌入：直接与序列中的特定位置相关联。

相对位置嵌入：关注词元之间的相对位置或距离。便于模型能够更好地适应不同长度（包括在训练过程中从未见过的长度）的序列。

OpenAI 的 GPT 模型使用的是绝对位置嵌入，这些嵌入会在训练过程中被优化，有别于原始 Transformer 模型中的固定或预定义位置编码。

**LLM 工作原理理解**：LLM 通过对大量无标注文本进行统计、预训练，将预测的句子中的下一个词作为“标签”；随后在更小规模且经过标注的目标数据集上进行微调，以遵循指令和执行分类任务。LLM 是基于 Transformer 的架构，其核心组件是注意力机制，即逐词输出时，根据需要调整输入序列的各部分的权重。

### **笔记**：

1.  微调大模型最流行的两种方法是**指令**微调和**分类任务**微调。不同微调类型有其相应的标注数据集：
    
    -   指令微调 —— “指令-答案”对
        
    -   分类任务微调 —— “文本及其类别标签”
        
2.  Transformer 基础架构  
    最初是为了机器翻译任务开发。
    
    -   编码器：负责处理输入文本，将其编码为一系列数值表示或向量，以捕捉输入的上下文信息。
        
    -   解码器：接收编码向量，并据此生成输出文本。
        
    
    编码器和解码器都是由多层组成，这些层通过**自注意力机制**连接。
    
    -   自注意力机制允许模型衡量序列中不同单词或词元之间的相对重要性。
        
    -   它可以很好地捕捉、缓解输入数据中长距离的依赖和上下文关系。
        
3.  Transformer 变体
    
    -   **BERT** (Bidirectional Encoder Representations from Transformer, 双向编码预训练 Transformer)：专注于掩码预测（masked word prediction），用于情感预测、文本分类等任务，比如以前推特利用 BERT 检测有害内容。
        
    -   **GPT**(Generative Pretrained Transformer, 生成式预训练 Transformer)：侧重于原始 Transformer 架构解码器部分，主要用于处理生成任务和生成文本序列，比如机器翻译、文本摘要、小说写作、代码编写等。最开始主要被设计和训练用于**文本补全**（text completion）任务，但它们表现出了出色的**可扩展性**。这些模型擅长执行**零样本学习任务**（在没有任何特定示例的情况下，泛化到从未见过的任务）和**少样本学习任务**（从用户提供的少量示例中进行学习）。
        
    
    \- GPT 本质上只包含解码器部分，并不包含编码器。由于像 GPT 这样的解码器模型是通过逐词预测生成文本，因此它们被认为是一种\*\*自回归模型\*\*（autoregressive model）。自回归模型将之前的输出作为未来预测的输入，提高了最终文本的一致性。
    
4.  **注意**：并非所有的 Transformer 都是大语言模型，因为Transformer 也可用于计算机视觉领域。同样，并非所有的大语言模型都基于 Transformer 架构，因为还存在基于循环和卷积架构的大语言模型。
    
5.  **词元**（Token）是模型读取文本的基本单位。
    
6.  这些类 GPT 模型的预训练特性使它们在针对下游任务进行微调时表现出了极高的灵活性，因此它们也被称为**基础模型**。预训练大语言模型需要大量资源，成本极其高昂。
    
7.  **自监督学习**（self-supervised learning）：用于下一单词预测任务，它是一种自我标记的方法，即可以使用句子或文档中的下一个词作为模型的预测标签。由于该任务允许“动态”创建标签，因此我们可以利用大量的无标注文本数据集来训练大语言模型。
    
8.  **涌现**（emergence）：模型能够完成未经明确训练的任务的能力。
    

### 问题回答

1.  为什么 LLM 使用无监督预训练而非直接监督训练？  
    LLM 使用无监督预训练（更准确地说是自监督学习）的核心原因在于数据规模与获取成本的巨大差异。高质量的人工标注监督数据极其稀缺且昂贵，而互联网上存在着近乎无限的免费原始文本。通过在海量无标注文本上进行“预测下一个词”的预训练，模型能够像海绵吸水一样，低成本地学习到人类语言的语法结构、逻辑推理能力以及极其丰富的世界知识，从而构建出一个强大的“通用基座”。如果直接使用少量昂贵的监督数据从头开始训练，模型不仅难以学到通用的语言规律，还极易对特定任务产生过拟合，完全失去“举一反三”的泛化能力。
    
2.  GPT 系列为什么选择解码器-only 架构？  
    GPT 系列选择纯解码器（Decoder-only）架构，是因为这种设计与自回归生成任务（即文本续写）的本质完美契合。与擅长文本理解的 Encoder（如 BERT）相比，纯 Decoder 通过因果掩码（Causal Mask）严格保证了模型只能根据前文预测后文，非常适合生成式任务；而与 Encoder-Decoder 架构相比，纯 Decoder 在工程实现和算力扩展（Scaling）上更为简洁高效。OpenAI 敏锐地发现，只要模型规模足够大，绝大部分自然语言处理任务（如翻译、问答、总结）都可以被巧妙地统一转换为“文本续写”问题，从而让这种看似简单的单向生成架构涌现出强大的通用解决能力。
    
3.  RLHF 的核心思想是什么？  
    RLHF（基于人类反馈的强化学习）的核心思想是将模型的输出与人类的价值观和真实偏好进行“对齐（Alignment）”。经过预训练的模型虽然掌握了海量知识，但它的目标只是“预测最可能的下一个词”，有时会生成看似合理但实际上带有偏见、有害或答非所问的内容。RLHF 的做法是：先让人类评估员对模型的多个回答进行排序优劣打分，用这些打分训练出一个模拟人类喜好的“奖励模型（Reward Model）”；然后，使用强化学习算法（如 PPO）让 LLM 在生成文本时去追求奖励模型的最高分。这样一来，模型就从“顺着话说”转变为了“生成有用、诚实且无害的回答”。
