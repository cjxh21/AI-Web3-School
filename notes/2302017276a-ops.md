\## 二、每日打卡（Check-in）

\### 怎么打

1\. 点击「去残酷共学打卡（打开笔记编辑器）」

2\. 写学习笔记——记录今天学了什么、做了什么

3\. 提交

\### 规则

| 项目 | 说明 |

|------|------|

| 学分 | 每次打卡 **+40 学分** |

| 淘汰线 | **单周缺卡超过 2 次 → 淘汰**（红色边框 = 预警） |

| 水笔记 | 会被判定「未学习」，不算打卡 |

| 公开性 | ⚠️ 打卡笔记会公开在 GitHub，\*\*不要写密码/API Key/私钥\*\* |

| 时区 | 影响打卡判定，建议设对时区 |

| 自由模式 | 自由打卡不计学分，仅作学习记录 |

| 刷新 | 打卡状态同步有延迟，可点「手动刷新今日打卡」 |

\### 打卡 Heatmap

\- 绿色：已打卡

\- 灰色：请假/未打卡

\- 红色边框：淘汰预警（本自然周第 3 次未打卡）

\## 三、任务提交（Task Submission）

\### 核心流程

\`\`\`

找到任务 → 点「提交」→ 填标题 + 正文/附件 → 提交 → 生成工单 → 审核 → 完成

\`\`\`

\### 任务状态

| 状态 | 含义 |

|------|------|

| 进行中 | 当前可提交 |

| 已提交 | 等待审核 |

| 已完成 | 审核通过 |

| 已失效 | 已过期 |

| 即将开始 | 未开放 |

\### 提交要求

\- **标题不能为空**，正文或附件\*\*至少填一项\*\*

\- 附件有数量上限（最多 N 个）

\- 附件支持：截图、文件、剪贴板图片

\- 可选「学习记录模式」（仅个人记录，不提交审核、不计学分）

\### 提交内容类型（根据任务不同）

| 类型 | 举例 |

|------|------|

| 公开链接 | GitHub repo、Notion、博客、X/Twitter 链接 |

| 截图 | 工具配置、运行结果、合约部署 |

| 链上记录 | 交易哈希、合约地址、区块浏览器链接 |

| 文字说明 | 概念解释、学习总结、比较分析 |

\### ⚠️ 绝对不要提交的内容

私钥、助记词、API Key、token、.env 文件、身份证号、手机号

\## 五、提交技巧

1\. **一任务一工单**：每个任务单独提交，不要合并

2\. **链接优于截图**：能用 GitHub/Notion/X 链接的优先用链接

3\. **说明 AI 参与度**：哪些是 AI 做的、哪些人工修改/验证了

4\. **失败记录也提交**：遇到问题、修正过程照样有价值

5\. **提交后关注工单状态**：在「求助与反馈」可追踪审核进度

# Agent 入门笔记

> 讲师：Draken 老师 · 日期：2026-05-19

## 1\. 什么是 AI Agent？

### 公式

```
AI Agent = LLM + 工具集 + 自主决策循环
```

### 核心循环

```
感知 (Observe) → 思考 (Reason) → 行动 (Act) → 反馈 (Feedback)
     ↑                                              |
     └──────────────────────────────────────────────┘
```

### 与传统 Chatbot 的区别

|   | Chatbot | Agent |
| --- | --- | --- |
| 交互模式 | 一问一答 | 多步自主执行 |
| 工具使用 | 无 / 手动切换 | 自动调用工具链 |
| 上下文 | 单轮 | 长期记忆 + 跨会话 |
| 任务复杂度 | 简单 Q&A | 复杂多步任务 |
| 典型例子 | ChatGPT 对话 | 自动写代码 → 测试 → 部署 |

### Agent 的关键组件

1.  **LLM** — 大脑，负责推理和决策
    
2.  **Tools** — 手和脚，执行具体操作（终端、文件、API、浏览器…）
    
3.  **Memory** — 记忆，短期（对话上下文）+ 长期（持久化偏好/事实）
    
4.  **Planning** — 将大任务拆解为可执行步骤
    
5.  **Orchestration** — 多 Agent 协作调度
    

## 2\. Hermes Agent 简介

### 是什么

-   开源 AI Agent 框架
    
-   支持 CLI / Telegram / Discord / 微信 等多平台部署
    
-   可扩展 Skill 系统 — 类似插件，按需加载专业能力
    
-   原生支持 MCP (Model Context Protocol) 工具集成
    

### 核心特性

-   多模型支持：Claude / GPT / DeepSeek 等
    
-   文件系统 + 终端 + 浏览器 + 搜索 等内置工具
    
-   Cron 定时任务（可做每日学习提醒）
    
-   Sub-agent 委托（复杂任务拆解并行执行）
    
-   持久记忆（跨会话记住偏好和知识）
    

### 架构概览

```
用户 (Telegram/etc)
    │
    ▼
┌──────────┐
│ Gateway  │ ← 消息路由
└────┬─────┘
     ▼
┌──────────┐
│  Agent   │ ← LLM + System Prompt + Tools
└────┬─────┘
     ▼
┌──────────┐
│  Tools   │ ← terminal / file / browser / MCP servers
└──────────┘
```
