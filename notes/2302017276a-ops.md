# Agent 入门笔记

> 讲师：Draken 老师 · 日期：2026-05-19

## 1\. 什么是 AI Agent？

### 公式

```
AI Agent = LLM + 工具集 + 自主决策循环
```

### 核心循环

```
感知 (Observe) → 思考 (Reason) → 行动 (Act) → 反馈 (Feedback)
     ↑                                              |
     └──────────────────────────────────────────────┘
```

### 与传统 Chatbot 的区别

|   | Chatbot | Agent |
| --- | --- | --- |
| 交互模式 | 一问一答 | 多步自主执行 |
| 工具使用 | 无 / 手动切换 | 自动调用工具链 |
| 上下文 | 单轮 | 长期记忆 + 跨会话 |
| 任务复杂度 | 简单 Q&A | 复杂多步任务 |
| 典型例子 | ChatGPT 对话 | 自动写代码 → 测试 → 部署 |

### Agent 的关键组件

1.  **LLM** — 大脑，负责推理和决策
    
2.  **Tools** — 手和脚，执行具体操作（终端、文件、API、浏览器…）
    
3.  **Memory** — 记忆，短期（对话上下文）+ 长期（持久化偏好/事实）
    
4.  **Planning** — 将大任务拆解为可执行步骤
    
5.  **Orchestration** — 多 Agent 协作调度
    

## 2\. Hermes Agent 简介

### 是什么

-   开源 AI Agent 框架
    
-   支持 CLI / Telegram / Discord / 微信 等多平台部署
    
-   可扩展 Skill 系统 — 类似插件，按需加载专业能力
    
-   原生支持 MCP (Model Context Protocol) 工具集成
    

### 核心特性

-   多模型支持：Claude / GPT / DeepSeek 等
    
-   文件系统 + 终端 + 浏览器 + 搜索 等内置工具
    
-   Cron 定时任务（可做每日学习提醒）
    
-   Sub-agent 委托（复杂任务拆解并行执行）
    
-   持久记忆（跨会话记住偏好和知识）
    

### 架构概览

```
用户 (Telegram/etc)
    │
    ▼
┌──────────┐
│ Gateway  │ ← 消息路由
└────┬─────┘
     ▼
┌──────────┐
│  Agent   │ ← LLM + System Prompt + Tools
└────┬─────┘
     ▼
┌──────────┐
│  Tools   │ ← terminal / file / browser / MCP servers
└──────────┘
```
