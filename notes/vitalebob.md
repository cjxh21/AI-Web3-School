# 2026-05-31
<!-- DAILY_CHECKIN_2026-05-31_START -->




| 方向 | AI 作用 | Web3 机制 |
| --- | --- | --- |
| Payment / Commerce | 理解需求、比价、发起支付意图 | Intent、Budget/Quote、可验证结算 |
| Wallet / Permission | 规划交易、解释风险 | Session Key、Policy、多签、审计 |
| Identity / Capability | 能力匹配、编排 Agent | Registry、能力声明、MCP/A2A |
| Privacy / Security | 防注入、脱敏、沙箱 | 本地签名、最小权限、链上审计 |
| Dev Tooling / Workflow | 读 docs/合约、生成可视化 | 链上 trace、可复现记录 |
| Governance | 提案摘要、行动项 | 投票、金库、公开贡献记录 |

**A. Payment / Commerce（Payment Intent）**

-   不是纯 AI：AI 能代填单，但**不能单独建立**可追责的付费与履约闭环。
    
-   不是纯 Web3：链能转账，但**不知道买什么、是否在 Budget 内**——需要 AI 理解任务与 Quote。
    

**B. Dev Tooling — Research Visualization**

-   不是纯 AI：图表若无**可验证数据来源**，对外叙事缺公信力。
    
-   不是纯 Web3：链上 raw 数据**不会自动变成**洞察与可视化，仍要 AI 归纳解释。
