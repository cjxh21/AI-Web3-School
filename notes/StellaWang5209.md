# 2026-05-27
<!-- DAILY_CHECKIN_2026-05-27_START -->
Claude Code 不是传统意义上的代码补全工具，而更像一个可以实际参与开发流程的“AI代理”。相比之前用 ChatGPT 这种问答式工具，它不仅能回答问题，还能读取代码、执行操作并持续推进任务，因此使用方式也完全不同。

它的工作逻辑，是先获取上下文，再执行操作，最后验证结果。效果好坏很大程度取决于是否提供了清晰的上下文以及是否合理引导过程，而不是单次提问是否“问得好”。另外，Explore、Plan、Code、Commit 这一流程非常关键，它强制先让 AI 理解和规划，再动手实现，从而减少错误。

还有上下文管理的重要性，如果信息混乱或过载，结果就会明显变差。整体来看，使用 Claude Code 更像是在带一个初级工程师，而不是单纯使用工具。

# 2026-05-21
<!-- DAILY_CHECKIN_2026-05-21_START -->





当前 NLP 的核心已经从面向单一任务的模型，转向以大规模预训练语言模型为基础的通用能力模型。这类模型通过在海量文本数据上进行训练，学习语言的统计规律与语义结构，从而能够在不同任务之间实现迁移与复用，这种“预训练—微调”或“直接推理”的范式改变了传统模型开发的基本思路，现代 NLP 的本质已经逐渐演变为对统一语言表示能力的建模。

任何自然语言在进入模型之前，必须经过分词转化为离散的符号序列，并进一步映射为数值形式，这一过程本质上是将连续语义信息离散化并结构化表达。

模型并不是直接输出“答案”，而是基于输入上下文计算下一个词或一系列词的概率分布，不同任务（如分类、生成、翻译）实际上都是对这一概率分布进行不同方式的解释与约束。

模型参数规模的扩大虽然能够显著增强表达能力，但同时也带来了显存占用高、计算成本大的问题，因此需要通过量化等方法降低参数精度，从而在性能与资源之间取得平衡。

大模型不仅是算法问题，同时也是典型的工程优化问题，其发展依赖于模型设计与计算资源利用效率的协同提升。
