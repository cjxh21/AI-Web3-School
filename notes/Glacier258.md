# 2026-05-27
<!-- DAILY_CHECKIN_2026-05-27_START -->
在之前接触web3的基础上进行了概念的复习和扫盲

1.  **Hardhat：前端友好的合约工程框架**  
    Hardhat 的核心优势在于与 JavaScript/TypeScript 生态的无缝融合，非常适合需要前后端紧密协作的项目。学习它的重点不在于死记命令，而是理解它如何通过本地链、测试网、环境变量和部署脚本，将零散的开发动作串联成一套标准化的工程流程。
    
2.  **Foundry：追求极致效率与安全测试的原生工具**  
    Foundry 主打纯 Solidity 开发和命令行工作流，以极速编译和强大的测试能力见长。它特别适合用来训练“先写测试，再改合约”的习惯，通过 Fuzz（模糊测试）和 Fork（主网分叉测试），能在安全敏感的合约开发中提供比 UI 点击更可靠的保障。
    
3.  **OpenZeppelin：提供安全基石，但无法替代业务决策**  
    OpenZeppelin 提供了经过审计的标准模块（如 ERC-20、权限控制等），能大幅减少重复造轮子和基础漏洞。但开发者必须警惕“用了库就绝对安全”的错觉，因为谁能铸造代币、是否允许暂停、升级权限如何分配等核心产品逻辑，依然需要开发者自己审慎设计。
    
4.  **viem / wagmi：专注解决前端与链交互的状态同步**  
    viem 提供底层的类型安全接口，wagmi 则面向 React 提供便捷的 Hooks。在前端接入时，最容易踩坑的是状态混乱，因此必须在代码中将钱包连接、网络切换、数据加载以及交易生命周期（等待签名/广播/确认/失败）严格区分开来，才能给用户清晰的反馈。
    
5.  **实战串联：让错误尽早暴露的标准化流水线**  
    一个成熟的 DApp 开发流程应该是：先用 Remix 快速验证语法，再迁移到 Hardhat/Foundry 编写严密的单元测试和本地部署，接着用 viem/wagmi 搭建前端并精细处理各种异步状态，最后在测试网完成源码验证。这种看似繁琐的流程，能有效避免上线后出现地址错误、ABI 不匹配或交易状态卡死等致命问题。

### **让 Agent 拥有持续上下文与长期一致性**

_很重要的分享 此前还在做记忆体 每次都要让它总结 再带去下一对话_

1.  **记忆不是用来“存”的，是用来“长脑子”的**  
    给 Agent 加长期记忆，不能只是像用U盘一样存个聊天记录。真正的记忆得能帮它总结经验教训，让它下次遇到类似问题时变得更聪明。
    
2.  **“死记硬背”——>“分类整理”**  
    好的记忆系统会像人脑一样分工：有的负责记当下的事（短期），有的负责记用户喜好和知识（语义），还有的负责记住“怎么做某件事”的标准流程（程序性）。
    
3.  **好记性不如烂笔头，但也要学会“断舍离”**  
    虽然要记东西，但不能什么都往里塞。Agent 必须懂得过滤废话、压缩重点，不然装了一肚子没用的垃圾信息，反而会影响它的判断力。

# 2026-05-21
<!-- DAILY_CHECKIN_2026-05-21_START -->





今日学习：

1\. Vibe Coding—— “怎么干”

-   **核心定义**：由 Andrej Karpathy 提出，是一种**“意图驱动”**的开发模式。你不再纠结于语法和代码细节，而是专注于描述“想要什么感觉（Vibe）”或“实现什么目标”。
    
-   **角色转变**：开发者从“代码搬运工”升级为**“产品指挥官”**。你负责审核结果、提供反馈，AI 负责具体的编码和调试。
    
-   **典型场景**：快速构建 MVP（最小可行性产品）、非程序员通过自然语言生成网页或脚本。
    

2\. Frameworks—— “在哪干”

-   **核心定义**：框架是为了解决特定问题而提供的**半成品架构**。它规定了代码的组织结构（如 MVC 模式）和运行流程（控制反转）。
    
-   **在 AI 时代的意义**：虽然 AI 能写代码，但它容易写出“意大利面条式”的混乱代码。框架提供了**约束和规范**，确保 AI 生成的代码符合工程标准，易于维护和扩展。
    
-   **常见例子**：Web 开发中的 React、Next.js；AI Agent 开发中的 LangChain、CrewAI。
    

3\. MCP (Model Context Protocol) —— “连什么”

-   **核心定义**：由 Anthropic 推出的开源协议，被称为**“AI 的 USB-C 接口”**。它标准化了大模型与外部数据源（本地文件、数据库、企业 API）的连接方式。
    
-   **解决痛点**：以前，想让 AI 连接每个新工具都需要定制开发（m 个模型 × n 个工具 = m×n 次开发）。有了 MCP，只需开发一次服务器端，所有支持 MCP 的 AI（如 Claude Desktop）都能直接连接。
    
-   **核心价值**：**打破数据孤岛**。让 AI 不仅能“聊天”，还能实时读取你的本地文件、查询公司数据库或操作第三方软件。

<!-- DAILY_CHECKIN_2026-05-19_START -->






1.  **Prompt（提示词）是“沟通指令”**：它是人与 AI 交互的桥梁。本质就是你给 AI 下达的明确指令（包含角色、任务、约束条件等），用来引导 AI 生成你想要的特定内容或完成具体任务。
    
2.  **Workflow（工作流）是“固定流水线”**：它是一套预先设计好、严格按顺序执行的自动化流程。就像工厂的装配线，每一步（节点）和先后逻辑（连线）都是固定的，主打标准化、可重复和高效率。
    
3.  **Agent（智能体）是“自主打工人”**：它是以大语言模型（LLM）为“大脑”，并配备了记忆、规划和工具使用能力的自主系统。它不再只是被动等待指令，而是能主动拆解目标、调用工具，并在遇到变化时自我调整。

时间有限看了llm的基础概念 总结理解：

1.  **本质是“概率预测机”**：LLM 就像一个读遍了互联网书籍的超级学霸，它并不真正“理解”世界，而是靠统计规律疯狂猜你下一句想听什么。
    
2.  **思考方式是“文字接龙”**：它生成回答的过程，其实就是基于你给的前文，一个字一个字地往外蹦，玩一场高配版的“文字接龙”游戏。
    
3.  **核心强项是“文科生”**：它非常擅长写文案、润色邮件、头脑风暴和翻译，因为它精通语言的套路和模式，但不一定懂严谨的事实。
    
4.  **致命弱点是“一本正经地胡说八道”**：当它遇到知识盲区时，为了把句子编圆满，会自信地输出错误答案，这就是著名的“AI幻觉”。
    
5.  **技术骨架叫“Transformer”**：这是它最底层的架构，赋予了它强大的“注意力机制”，让它能搞懂一句话里谁和谁是有关联的（比如知道“他”指的是谁）。
    
6.  **知识是有“保质期”的**：它脑子里的知识只停留在训练数据截止的那一天，不知道那之后发生的新闻，除非给它配上联网搜索等“外挂”。
