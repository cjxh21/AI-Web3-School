今天写了篇推文  
**LLM：大语言模型**

它就像一个人类，从小到大读了几万亿字的书，什么都见过一点点。你问它"感冒了怎么办"，它不是现场想答案，而是从记忆里提取出一个答案

_LLM 就是这样：它不是真的懂道理，而是见过太多例子，知道该怎么给出回复_

## **Prompt：提示词**

想象一下：你跟学霸说"帮我个忙呗"，TA不知道你要什么。但你具体地说"帮我写一封请假邮件，语气正式一点，我要交给项目经理"，它就能给你写出来

**Prompt 就是你跟 AI 说话的方式** — 你说得越清楚、限定条件给得越多，它答得越准确。同样一个 AI，换个问法，效果可能完全不一样

> 一个好的 Prompt = 把 AI 当成一个不了解你情况的助理。你需要把背景、要求、格式都交代清楚

## **Context Window：上下文窗口**

学霸做题，草稿纸只有一张那么大。你在这张纸上写的东西越多，之前写的东西就得擦掉挤掉

LLM 也一样，它一次能处理的内容是有限的 ，这个限制就叫 context window

_你可以理解为 AI 的"脑子一次能装多少信息"，也就是我们常说的“脑容量”_

有的模型能处理 20 万 token ≈ 15 万汉字，有的只有 8K ≈ 6 千字。超出的部分它就"忘了"

> 所以 Chat页面 里为什么不能说太多？因为草稿纸不够大了

## **Workflow：工作流**

你想做一顿饭，可"做一顿饭"不是一步到位的哦。它需要有一个标准的SOP：

1.  打开冰箱 → 看有什么菜
    
2.  决定做什么菜
    
3.  洗菜切菜
    
4.  炒菜
    
5.  装盘
    

每一步依赖上一步的结果，这就是一个 workflow。

AI 里的 workflow 也是一样

_把一个复杂任务拆成_**_一连串有先后顺序的步骤_**_，前一步的输出自动变成后一步的输入，自动化跑完整个流程_

> 用 AI 写营销文案的工作流 = 选品 → 分析竞品评论 → 提炼痛点 → 生成文案 → 翻译英文

## **Agent：智能体**

普通的 AI 就是你问它答，问一次答一次。你情不自禁地发问：你怎么跟人机一样。——嘿，它可不就是人机吗？？？

Agent 不一样 你告诉它一个目标，它自己拆解步骤、调用工具、判断什么时候做、什么时候停。_就像一个会做事的成年边牧_（边牧：勿cue）

-   **普通 AI** = 你说"帮我请半天假"，它说"好的，以下是请假模板"
    
-   **Agent** = 你说"帮我把下周会议都安排好"，它自己去翻日历、发邮件、确认时间、给你汇报结果
    

_Agent = 目标 + 自主行动能力_

## **Tool Use：工具调用**

Tool Use就是一本 **使用教材**

学霸脑子再好，你让他算 500 位数的乘法，他用手算还是会错

但如果你给他一个计算器，他就能算得又快又准

Tool Use 就是这个意思 — **LLM 本身不擅长实时查询、数学计算、发送邮件等操作，但如果它能调用外部工具，这些能力就能补上。**

> Tool Use = 让 AI 长出手脚，能做事而不只是说话

## **AI Coding：AI 编程**

传统的 coding 是你自己写代码，搜索引擎是字典，Stack Overflow 是问答论坛

AI coding 不一样 — _它能直接帮你写代码、改代码、解释代码、找 bug_ 那么这个时候就有人要问了：主播主播，你的AI coding确实很强，但还是太吃操作了。有没有更加简单又强势的coding推荐一下吗？

有的兄弟，有的，像这么强势的coding当然不止一个啦

那就是 **Vibe Coding**

**「我要做一个抖音，帮我搞定」**

Vibe Coding，你只管说出你想做什么，用自然语言不断调整，AI 负责把代码写出来、改出来。你基本不碰代码，就在那儿 "vibe"（凭感觉）指挥，适合萌新小白使用。本人经常用的 vibe coding 就是 **OpenClaw 、Cursor**

## Web3 运行原理

* * *

### 一、核心逻辑

Web3 的本质是：**用一群互不相识的人共同维护一个数据库，没有人能单独控制它。**

这个数据库叫**区块链**，每个参与者（节点）都存一份完整副本。任何改动必须被大多数人同意才能写入，且写入后无法更改。

* * *

### 二、三大技术支柱

```
区块链（去中心化账本）
    ↓
智能合约（自动执行的程序）
    ↓
代币经济（激励机制）
```

* * *

### 三、工作流程（以转账为例）

```
A 给 B 转 1 ETH
    ↓
交易被广播到全网所有节点
    ↓
矿工/验证者验证：A有足够余额 + 签名有效
    ↓
交易被打包进新区块
    ↓
全网节点同步更新账本
    ↓
B 收到 1 ETH（不可逆）
```

* * *

### 四、为什么去中心化？

| 中心化（传统互联网） | 去中心化（Web3） |
| --- | --- |
| 数据存在一家公司服务器 | 数据存在全球所有节点 |
| 公司可以删帖、封号、冻结资产 | 规则写在代码里，没人能篡改 |
| 你的数据是平台的资产 | 你的资产真的属于你（私钥） |

* * *

### 五、智能合约如何工作？

```
传统合同：A欠B钱 → 找律师/法院强制执行
智能合约：A给B转钱 → 代码自动执行，无需任何人
```

代码即法律，触发条件满足，自动执行，不存在违约可能。

* * *

### 六、Web3 的盈利逻辑

| 方式 | 说明 |
| --- | --- |
| 炒币 | 买卖加密货币赚差价（零和博弈） |
| DeFi | 借贷、流动性挖矿、质押（基于真实收益） |
| NFT | 数字资产确权与交易 |
| Web3 就业 | 开发、设计、运营、BD |
| 撸空投 | 项目方免费撒钱获取早期用户 |

* * *

### 七、Web3 与 AI 的结合

```
Web3 解决：数据确权 + 价值传递 + 去中心化
AI 解决：信息处理 + 智能决策 + 效率提升
```

本次学习内容是“AI时代的Web3架构能力”

```
Web3 × AI = 去中心化AI基础设施
         ↓
    数据层（确权）→ 模型层（训练/推理）→ 应用层（交互）
```

* * *

### 各层能力拆解  
1\. 数据层——AI 数据的确权与交易

| 能力 | 说明 |
| --- | --- |
| 数据 NFT 化 | 将 AI 训练数据打包成 NFT，确权后可在市场上交易 |
| 数据 DAO | 通过 DAO 治理结构，集体贡献数据并共享收益 |
| 隐私计算 | 使用零知识证明（ZKP）、同态加密，让 AI 模型在加密数据上训练，原始数据不泄露 |
| 联邦学习 | 各节点本地训练模型，参数上链聚合，数据不出本地，满足隐私合规 |

**代表项目**：Ocean Protocol（数据代币化）、Filecoin + AI（分布式存储训练数据）

* * *

2\. 模型层——去中心化 AI 模型训练与推理

| 能力 | 说明 |
| --- | --- |
| 模型 NFT | 将训练好的模型参数铸造成 NFT，明确产权归属 |
| 算力去中心化 | 利用全球闲置 GPU 算力（如 Render Network、Flux）训练 AI 模型，降低训练成本 |
| 模型推理上链 | 智能合约调用链上/链下 AI 推理，结果可验证、可追溯 |
| AI Agent 自主执行 | AI Agent 持有私钥（钱包），根据链上事件自主执行交易/操作，无需人工介入 |

**代表项目**：Render Network（GPU 渲染/AI 计算）、[Fetch.ai](http://Fetch.ai)（AI Agent 经济）、SingularityNET（AI 模型市场）

* * *

3\. 应用层——AI + Web3 的实际场景

| 能力 | 说明 |
| --- | --- |
| AI 生成内容（CAI）+ NFT | AI 生成的图片/音乐确权后直接 mint 为 NFT，版权与收益自动分配（版税归属） |
| DeFi + AI | AI 提供智能投研、反欺诈、信用评分；区块链提供透明清算与可验证性 |
| GameFi + AI NPC | AI 驱动的 NPC 拥有自主意识，决策上链，玩家的游戏资产真正属于自己 |
| DAO + AI 治理 | AI 辅助投票分析、提案风险评估、议事流程自动化，降低参与门槛 |

* * *
