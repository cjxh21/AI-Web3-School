今天的分享会有点高端，主题是"新密码朋克与隐私的文化层次"。  
主要内容包括：

1.  新密码朋克的定义：新密码朋克（Neo-Cypherpunk）不是对90年代的怀旧，而是隐私、玩乐与社区的融合，是一个人人皆可参与的现代运动，因为隐私与自主权属于所有人。
    
2.  核心理念：将隐私重新定义为公共利益，鼓励不同背景的人合作，以互助代替竞争，以积极快乐的态度而非恐惧来捍卫自由。

4.  新旧密码朋克的区别：旧密码朋克面向专家和密码学家，强调个人隐私；新密码朋克面向所有人，强调社区共同保护，注重共情、互助和开放对话。
    
5.  文化工具包：包括使用隐私优先的应用、链上原生工具、社区聚会交流、传播表情包和贴纸等方式推广隐私文化。

# 2026-05-26

MPC（多方计算钱包安全）

解决钱包底层安全问题，即私钥由谁持有

市面上其他agent大多依赖TEE或临时发放session token/API key的方式做授权，存在单点风险

Cobo的方案：Cobo、Agent、Human三方各自持有对应的密钥分片，任意一方都无法单独转移资金

采用2/2门限签名模式，分两个场景：

第一：Agent与Cobo共管（适用于人类批准了某个Pack后，Agent在授权范围内自动执行）

第二：Human与Cobo共管（适用于大额资金转移，由人类发起签名）

Pack（授权协议）

定义Agent能做什么、不能做什么

Pack的四个核心要素：

第一：Intent（意图）：期待Agent完成什么任务，例如"ETH低于2000时买入，高于2500时卖出"

第二：Execution Plan（执行计划）：AI将意图转译为具体执行步骤，包括调用哪个合约、执行数量等

第三：Policy（风控约束）：包括预算审批、白名单、合约限制、API参数级约束，会在每次执行时强制校验

第四：Completion Condition（完成条件）：Pack不能永久有效，设定时效或总额上限，超过后自动拦截

一、核心定义 & 基础认知
1.1 记忆本质
并非单纯扩充大模型上下文窗口
本质是跨时间的意图 - 状态管理系统
核心作用：跨会话传递有效状态，持续承接用户未完成的诉求与任务
1.2 两大分析视角
产品视角：聚焦用户交互体验的连续性
工程视角：侧重系统落地，包含记忆分类、检索逻辑、行为管控等
过渡特征：当记忆开始影响智能体行为时，就必须引入完整的生命周期管理
1.3 产品核心诉求
核心目标：避免用户在每次会话中重复搭建上下文
两种交互模式 
恢复模式：保留历史有效状态，接续原有工作，是理想形态
重启模式：清空所有状态，每次交互从零开始
二、典型产品实战案例
2.1 ChatGPT 双层记忆架构
持久化记忆：存储长期固定信息与用户偏好，支持手动编辑、删除
会话上下文：存放临时可变内容，根据当前对话场景按需调用
2.2 代码类智能体记忆（Claude Code）
核心目标：保障项目连续性（区别于通用对话产品的用户连续性）
需留存内容：项目构建 / 测试命令、团队开发规范、历史决策与踩坑记录
记忆能力演进三阶段 
手动维护：依靠本地笔记、配置文件管理项目信息
平台托管：通过内置功能统一管理项目记忆，简化重复配置
全自动管理：系统自动维护任务状态，无需人工介入
2.3 记忆与智能体自主性的关联
对话型智能体：记忆用于优化对话流畅度，减少重复沟通
工具，代码型智能体：记忆用于保障任务、项目稳定推进
高自主型智能体：记忆升级为行为控制层，直接决定后续动作
通用底层逻辑：有选择地持久化信息、有选择地调取内容、及时舍弃无效数据

三、核心分类模型：记忆二维正交轴
3.1 维度一：时间轴（按生命周期划分）
短期记忆（当前工作上下文） 
覆盖范围：单会话内临时内容、推理草稿
补充说明：推理草稿属于短期活跃状态，不归入内容分类体系
长期记忆（持久化记忆） 
覆盖范围：跨会话、跨任务长期留存的内容
3.2 维度二：内容轴（仅针对长期记忆）
参考心理学记忆分类，分为三类
情景记忆：记录过往交互事件、具体场景与经历
语义记忆：存储客观事实、用户偏好、静态知识库等信息
程序记忆：保存操作流程、业务规则、执行方法与技能
3.3 组合使用方式
可组合描述记忆类型，例如：长期语义记忆、长期情景记忆、短期工作上下文

四、易混术语深度辨析
4.1 Recall / Retrieve / Revise 三者区分
回忆：偏向用户体验层面，是用户感知到 “智能体记住过往内容” 的直观效果
检索：偏向系统操作层面，是定位、提取已有记忆的技术动作
修订：偏向系统维护层面，对旧记忆进行更新、合并、淘汰等操作
4.2 核心结论：记忆 ≠ 检索
检索：只是单一的信息查找环节
RAG：基于检索 + 大模型生成的工作流，不等同于完整的记忆系统
记忆：一套完整体系，包含信息存储、调取、更新规则，同时管控智能体行为
4.3 五层技术角色分层（自上而下）
记忆：顶层核心能力，实现跨时间状态留存
检索：核心执行动作，完成信息定位与提取
RAG：落地应用工作流，结合检索结果完成内容生成
向量嵌入：基础技术，将文本转化为向量格式
向量数据库：底层基础设施，负责向量存储与相似度搜索

五、记忆工程：全生命周期管理
5.1 四大闭环核心环节
选择性捕获 
核心规则：过滤闲聊、无效噪声，仅留存高价值信息，不做全量存储
选择性检索 
核心规则：不会调取所有存量记忆，仅返回与当前场景高度相关的内容
持续修订
主动遗忘 
核心规则：对过期、错误、失效的记忆进行标记、更新或清理
5.2 核心思想
记忆工程本质是全生命周期的综合治理
落地难点不在于 “如何存储数据”，而在于存、取、改、忘整套策略的设计
六、记忆在 Agent 技术栈中的位置
记忆贯穿整套技术架构，与三层模块深度融合
提示工程 
作用：将记忆作为上下文引用，不负责定义记忆持久化规则
上下文工程（记忆核心载体） 
作用：结合检索能力与记忆数据，决定大模型可见的全部信息
执行控制层 
作用：承载记忆修订、遗忘、运行监控、可观测性等能力
七、主流记忆生态项目
7.1 Mem0
定位：通用型记忆中间件
特点：支持跨框架、跨大模型使用，可快速插拔集成，提供标准化接口
7.2 Letta
定位：面向有状态智能体的专属记忆系统
特点：记忆是智能体核心运行状态的一部分，智能体可自主修改记忆，保障跨会话状态连续
7.3 EverOS
定位：面向智能体的记忆操作系统
特点：将长期记忆定义为底层基础设施，适配复杂、长期运行的智能体场景

八、记忆角色演进：从功能到基础设施
伴随智能体能力提升，记忆的定位逐级升级，共分为三个阶段
阶段 1：便利功能
适配场景：纯问答类智能体
核心作用：减少重复对话，优化使用体验
特点：即使缺失记忆，系统依旧可以正常运行
阶段 2：控制层
适配场景：具备工具调用、工作流执行能力的智能体
核心作用：约束智能体行为，保障操作安全、执行结果准确
风险提示：错误记忆会直接引发业务风险
阶段 3：基础设施
适配场景：企业级复杂智能体、多智能体协同系统
核心作用：支撑全系统运行连续性、多任务协同、行为可控
特点：可靠的记忆系统是整体服务正常运转的前提

智能体的行动能力越强，记忆在系统中的战略地位就越高
九、全局总复盘
9.1 两大主线
1.产品主线：体验优化 → 行为管控 → 底层基础设施
2.工程主线：术语区分 → 记忆分类 → 生命周期治理 → 全技术栈落地
9.2 落地避坑要点
1.不能将 RAG、检索功能等同于完整的智能体记忆系统
2.记忆需分层设计，区分短期 / 长期、情景 / 语义 / 程序记忆
3.记忆工程不只是存储与检索，更新、遗忘、常态化治理是关键
4.按需选型：通用场景选用通用记忆中间件，有状态智能体搭配专属记忆框架

自己总结：
AI × Web3 的方向不是“把 AI 放到链上”，而是让 AI Agent 在可授权、可结算、可审计的边界内使用链上能力；同时用 AI 降低 Web3 在理解、操作和风控上的门槛。
五个方向：去中心化AI基础设施，AIagent+钱包，AI链上分析数据，智能钱包和安全助手，交易所与DeFi智能风控
区块链是用来帮助AI进行算力计算，购买数据，调用API，发起交易，管理资产，会遇到账户支付授权，记录，追责问题

1.去中心化 AI 基础设施通常不是把大模型直接放到链上运行，而是链下计算，链上负责激励、结算、验证和服务发现
案例：
- Bittensor：用 subnet、miner 和 validator 组织不同类型的 AI 服务供给，让模型输出、推理服务或数据服务在各自任务中竞争，并通过评分和激励分配收益。
- Render / Akash：把 GPU、云计算和 AI/ML workload 放到更开放的资源市场中，尝试与中心化云形成对照。
- Filecoin / Arweave：为 AI 数据集、模型权重和长期记录提供存储路径。

2.AI Agent + 钱包
接入钱包和链上 action 之后，Agent 才能在授权范围内完成转账、swap、合约调用、部署或支付等动作
案例：
- Coinbase AgentKit：把 LLM 的工具调用连接到真实链上动作，让 Agent 能通过 wallet provider 完成签名和广播。
- Privy Agentic Wallets / MoonPay Agents：分别从授权体验、支付、稳定币消费和应用内钱包管理切入。
- Virtuals Protocol / elizaOS：代表消费端 Agent 经济体和开源 Web3 Agent 框架两类探索。

3.AI 驱动的链上分析工具
链上数据不好理解，普通用户看到的是地址、哈希、合约调用和跨链记录；AI 的价值在于清洗、标签、解释、生成假设，并把复杂数据转成可行动的判断
案例：
- Arkham：把地址层数据翻译成实体画像和资金流情报，用标签、聚类和归因帮助用户理解链上行为。
Nansen / Dune / Bubblemaps：分别从 Smart Money 标签、自然语言查询与 dashboard、钱包集群可视化等角度降低链上研究门槛。

4.智能钱包和安全助手
用户看不懂签名之后会发生什么。智能钱包和安全助手要解决的，是把“确认”变成“解释后果 + 识别风险 + 必要时拦截”。
案例：
- Blockaid：在用户访问 dApp 或发起交易时，结合域名、合约、calldata、交易模拟和风险模式，识别钓鱼、恶意授权和高风险交互。
- Rabby：通过交易模拟、风险扫描和授权管理，让用户在签名前看到资产变化和潜在风险。
- Safe / Coinbase Smart Wallet / Argent：从多签、账户抽象、passkey、social recovery、gas sponsor 等方向改造账户体验。

5.交易所和 DeFi 智能风控
交易所、协议、做市商、机构和合规团队能不能及时发现异常交易、攻击路径、清算风险和可疑资金流。
案例：
- Chainalysis / Elliptic：偏机构级链上合规和 KYT，重点是资金来源、制裁、诈骗、可疑行为和调查证据链。
- Forta / Hypernative：偏实时监控、攻击检测、交易前防护和自动化响应。
- Chaos Labs / Gauntlet：偏 DeFi 参数模拟、清算风险、市场健康度和持续建模。

### 🏗️今天把hermes和langGraph结合起来了

1\. 架构认知的觉醒：为什么要用 LangGraph？

在今天之前，你的 Hermes 是个“全能单体战神”，容易失控、容易死循环、发推特拦不住。引入 LangGraph 不是为了跟风，而是为了解决业务落地的三大痛点：

-   **消除黑盒（Determinism）：** 用拓扑图（Graph）锁死大模型的不确定性，让它必须按“抓取 ➡️ 风控 ➡️ 撰写 ➡️ 质检”的既定流水线工作。
    
-   **状态共享（State Management）：** 抛弃了变量传来传去的混乱，给整个流水线建了一个“共享云盘（State）”，所有 Agent 都在这一个盘里读写数据。
    
-   **强制刹车（Human-in-the-Loop）：** 这是金融和自媒体最核心的安全底线——在最终发布前，程序必须挂起，等待人类老板（你）点击确认。
    

### 🛠️ 2. 工程落地的三步走战法

咱们没有一上来就写复杂的图网络，而是用极其老辣的工程策略推进的：

-   **第一步：拆解 Skill（组件化）。** 把原来杂糅在 `if __name__ == "__main__":` 里的执行逻辑，封装成纯净的 `execute_financial_report_workflow()` 函数，并用 `sys.path` 解决了恶心的跨目录导包问题。
    
-   **第二步：巨无霸节点跑通（防弹测试）。** 先把整个工作流塞进一个 LangGraph 节点，证明 `Gateway -> 引擎 -> 底层函数` 的物理链路是通的，成功丢掉了原厂的黑盒 `hermes gateway` 命令，彻底由你的 `run_agent.py` 接管。
    
-   **第三步：6 节点企业级流水线（精细化重构）。** 把巨无霸大卸八块，设立了**情报员 (Fetcher)、分析师 (Analyst)、主笔 (Writer)、主编 (Editor)、人类老板 (Human) 和发布员 (Publisher)** 6 个标准岗位。
    

### 🧠 3. LangGraph 的四大核心魔法

你今天实打实写出来的代码，用到了 LangGraph 最精髓的四大武器：

1.  `TypedDict` **(全局状态字典)：** 定义了 `raw_news`、`draft_tweet` 等字段，是流转的公文包。
    
2.  `add_node` **&** `add_edge` **(打工位与传送带)：** 定义了各个函数节点，并将它们通过主干道连起来。
    
3.  `add_conditional_edges` **(智能岔路口/循环打回)：**
    
    -   **过滤机制：** 没有好新闻直接结束（下班）。
        
    -   **循环机制（神来之笔）：** 主编查字数，如果不合格，把箭头反向指回主笔，强迫重写（打破了大模型话痨的死局）。
        
4.  `MemorySaver` **&** `interrupt_before` **(记忆与拦截)：** 引入 SQLite 级别的状态持久化，让程序能在“等老板确认”时安全暂停，甚至重启服务器后都能接着跑。
    

### 💡 4. 终极思想升华：规则路由 vs. 语义路由

这是今天最有含金量的反思。你极其敏锐地发现：**咱们现在虽然用了 LangGraph，但岔路口的判断（比如查字数）还是用 Python 写死的** `if/else`**。**

-   **妥协的艺术：** 因为像“精确数字数”这种物理边界问题，大模型算不准，用传统代码写死是最安全的。咱们借用 LangGraph，主要是馋它的“状态持久化”和“人工暂停”功能。
    
-   **未来的方向（LLM Router）：** 真正的 Agentic Workflow，岔路口应该由另一个“裁判大模型”来做决定（比如让大模型判断“这篇推文情绪对不对”）。这为咱们下一步让系统变得“更聪明”留下了伏笔。
