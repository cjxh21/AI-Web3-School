# 5/18(一)課程筆記

### 核心觀念：AI 時代的開發者定位

-   **基礎知識的不可取代性**：有了 AI，基礎知識反而更重要。AI 能大幅提升 Coding 與學習效率，放大個人能力，但**系統複雜度、錯誤成本與安全邊界並沒有改變**。
    
-   **人的價值定位**：人負責設計架構，AI 協助細化與執行；人必須擔任 AI 產出的「審查者」。開發者若缺乏紮實的基礎知識與架構能力，將無法判斷 AI 生成的程式碼是否正確（安全的前提是基礎紮實，否則都是空中樓閣）。
    
-   **結論**：我們要駕馭 AI（將其作為放大器），而非被 AI 驅動，最終的審核與安全責任仍在開發者手中。
    

### Web2 與 Web3 的融合與差異

-   **並非互斥，而是共生**：Web3 不是完全獨立於 Web2。多數 Web3 專案（如交易所 Binance、預測市場 Polymarket）仍重度依賴 Web2 的前端頁面與後端管理技術，Web2 的經驗可大量複用。
    
-   **底層信任機制的轉換（以支付系統為例）**：
    
    -   **Web2 (Bank Transfer)**：透過 API 交互，信任基礎在於 KYC、密碼權限與「中心化機構後端 (Institutional Backend)」。
        
    -   **Web3 (穩定幣支付)**：將銀行機構替換為 Blockchain，不同公鏈形同不同銀行。信任基礎轉移為：透明度 (Transparency)、不可篡改性 (Immutability)、共識機制 (PoW, PoS) 以及私鑰簽名。
        

### Web3 核心命題：安全與資產保護

在 Web3 中，涉及大量資金交互，「安全」不是事後附加的插件，而是貫穿產品生命週期的核心屬性。在 Web3，**對安全的理解力等同於資產保護力**。

1.  錢包與身份
    

-   **定義**：錢包在 Web3 中代表用戶的「身份」與「資產管理」憑證，用以證明用戶行為。
    
-   **私鑰與簽名**：私鑰是透過數學算法保證的大數，運作邏輯為`Message -> Input Private Key -> Output Signature`。
    
-   **錢包分類與託管**：
    
    -   類別：EOA (公私鑰對)、Smart Contract Wallet (智能合約錢包，如 Safe 多簽)、AA (帳戶抽象，結合 EOA 與智能帳戶)。
        
    -   託管方式：自託管、全託管 (如交易所)、混合託管 (Hybrid)。
        
-   **錢包安全防線**：嚴防私鑰外洩 (Private Key Leak)、防範簽名欺騙 (如禁用 `eth_sign`、提高簽名可視性)、防止權限濫用。
    

2.Web3 交易週期與關鍵參數

交易週期：**構造 -> 簽名 -> 廣播**

-   **Gas (交易手續費)**`Gas Fee = Gas Count * Gas Price` (受 EIP-1559 影響)。
    
-   **Nonce (確保交易冪等性，防重放攻擊)**：
    
-   ETH (Account 模型)：使用預設 Nonce 排序。
    
-   Tron/Solana：無 Nonce，透過 SDK 使用交易 Hash 確保。
    
-   BTC (UTXO 模型)：Cash 只能用一次，原生具備冪等性。
    
-   **Calldata**：鏈上交易的靈魂，可視為 Interpreter，執行端會自行解析交易 Data  
    

3.交易安全實踐

-   **交易模擬** ：在提交交易前驗證執行結果，且**必須在「簽名」前進行模擬**。
    
-   **Server 端安全**：資金量拆分以減少單點風險；利用可信任執行環境 (TEE) 保護私鑰，確保無任何明文暴露。
